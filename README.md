# Dynamic Beamer Presentation Monorepo

This repository uses a monorepo architecture to manage multiple dynamic Beamer presentations that leverage Python-generated TikZ animations.

## Repository Structure

- `oleron/`: Main seminar presentation on Optimistic/Pessimistic algorithms.
- `linucb/`: 3D demonstration of the LinUCB algorithm.
- `maze/`: Isolated project for the maze solver RL visualization.
- `shared_assets/`: Images and assets shared across multiple presentations.
- `pyproject.toml`: Global Python dependencies managed by `uv`.

## 🛠 Compilation Requirements

### Shell Escape
Strictly required for Python script execution via `\input|"..."`. Always compile using the `--shell-escape` flag:
```bash
pdflatex -shell-escape beamer.tex
```

### 📺 Viewing Animations (Blank PDF Trap)
The `animate` package uses embedded JavaScript for interactivity. Most standard PDF viewers (macOS Preview, Chrome, VS Code internal viewer) will display a **white screen** or a static image instead of the animation.
- **Linux:** Use **Okular** (Recommended).
- **Windows/macOS:** Use **Adobe Acrobat Reader**.

## ⚠️ Common Traps & Troubleshooting

### Undefined Axis Error
The `\begin{axis}` environment (from `pgfplots`) must **always** be wrapped inside a `tikzpicture` environment. Raw axis code generated by Python scripts will crash LaTeX if not enclosed:
```latex
\begin{tikzpicture}
    % Python generated axis here
\end{tikzpicture}
```

### Nullfont Warnings
If you see numerous `nullfont` warnings (`Missing character: There is no ; in font nullfont!`), check for trailing semicolons (`;`) on non-drawing TeX macros (like `\pgfmathsetmacro` or `\color`) inside TikZ blocks. TeX tries to "print" the semicolon but since it's in a path-less context, it uses `nullfont`.

### Architecture Best Practices
- **Avoid Monolithic Documents:** Keep complex animations in isolated scripts or separate `.tex` files.
- **Drafting Mode:** Use low sample rates in your Python generators during development to speed up compilation.
- **Performance:** Avoid heavy inline high-performance math loops (like 3D surface generation) directly in the main `.tex` file; delegate these to Python.
- **Preamble DRY:** Keep the preamble clean. Audit package imports to ensure each dependency is loaded exactly once with consistent options.

## 🚀 Getting Started

Each project folder contains its own `Makefile`. To compile a presentation:
1. Ensure `uv` is installed globally.
2. Navigate to the project folder (e.g., `cd oleron`).
3. Run `make`.

The root `pyproject.toml` defines all required Python dependencies. The virtual environment is managed at the repo root level for shared consistency.
